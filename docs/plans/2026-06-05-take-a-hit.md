# Take a Hit (Slice v4) - Implementation Plan

Status: Ready for implementation (2026-06-05) - created after plan review; missing plan file and spec-to-codebase gaps closed here.

Review findings closed:
- The requested plan file was absent; this document is the execution plan for `docs/specs/spec-take-a-hit.md`.
- JSON Schema alone cannot generally enforce `retaliation_damage.max >= min`; the plan adds a `validate_shared.py` cross-check and Go loader/runtime tests for that invariant.
- Dead-player authority must cover both explicit intents and already-active movement, so `applyMovement` also needs a dead-player guard.
- Resume-player-death is mostly a client snapshot rule in this slice; the default resume path reconstructs the sim from seed plus persisted inventory, so the plan verifies the client branch with a focused headless harness in `test_animation.gd` (not default smoke).
- `tools/validate_shared.py` already scans `shared/assets`; only retaliation golden/rule cross-checks are new.
- **Golden HP math corrected:** `pinned_seed` `deadbeefdeadbeef` kills the dummy in **one** hit (not three), so `final_player_hp` is **9** (`10 - 1` retaliation). Bot/smoke use random seeds and assert `hp < 10`, not the exact golden value.

Goal: Prove bidirectional combat by letting the training dummy retaliate on successful player hits, with server-authoritative player damage events and local-player `hit` / `death` animation reactions.

Architecture: Add optional per-monster retaliation damage to shared rules; the Go sim applies retaliation inside `handleAttack` after successful monster damage and emits player events through the existing `state_delta.events` array; the Godot client maps player events to existing `AnimationController` one-shot/terminal clips. No protocol envelope bump.

Tech stack: Go deterministic sim/tests, JSON Schema + Python shared validation, Godot 4.6.3/GDScript animation/smoke, Python protocol bot.

Spec: `docs/specs/spec-take-a-hit.md`
Baseline: slice v3 `animate-and-react` (complete - `make ci` green)
Branch: `feature/take-a-hit` (to be created)
