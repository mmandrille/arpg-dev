## 1. Executive summary

**Overall score: 8.1 / 10.**

This is a genuinely impressive solo/AI-driven project. In 53 vertical slices it has built a deterministic, authoritative ARPG with a clean client/server boundary, cross-language golden-fixture parity (Go ↔ GDScript), seeded-RNG replay, multi-level dungeon generation, co-op sessions, a full itemization/economy stack, and an end-to-end CI gate (`make ci`) that runs Go tests, headless Godot unit/smoke, a protocol bot, a Godot client bot, replay verification, and shared-schema validation. The **process discipline is the standout**: 1:1 spec→plan→slice→golden→CI, enforced determinism conventions, and rules-as-data with no logic leaking into JSON. Very few hobby-scale game projects reach this rigor.

The **architecture is sound at the boundaries but rotting at the centers.** The same anti-pattern repeats in every layer: a single god-file absorbs each new slice instead of the codebase decomposing as it grows.

| Artifact | Lines | Role | Verdict |
|----------|-------|------|---------|
| `server/internal/game/sim.go` | **7,036** | Authoritative simulation | God object — ~16 concerns, 329 methods on `*Sim`, no handler registry |
| `client/scripts/main.gd` | **4,410** | Client coordinator | God object — net + entities + input + camera + 12 UI panels + bot/replay scaffolding |
| `tools/bot/run.py` | **4,824** | Protocol bot | Monolith — driver + state + assertions in one importable-as-whole file |
| `tools/validate_shared.py` | **2,779** | Shared validator | God script — schema + rules + formula-sim fused, one 2,600-line function |
| `server/internal/game/game_test.go` | **7,144** | "Everything else" tests | Grows monotonically; navigable only by grep |

None of these are *broken* — they're well-named, mostly typed, and covered by tests. But every one of them is a place where "add the next slice" means "edit the 5,000-line file," which is exactly the friction that compounds into a wall. The domains-ai review you used as the template scored 9.7 **specifically because it had already split its 2,561-line monolith into packages**. This repo has not yet taken that step, and its monoliths are 2–3× larger.

**The encouraging part:** the boundaries are clean enough that decomposition is mechanical, not architectural surgery. The sim imports no I/O. Panels talk to `main.gd` over typed signals, not back-references. Rules are data. The hard design work is done; what remains is carving.

## 3. Cross-cutting themes (the real story)

### Theme 1 — The god-file tax is now the #1 threat to velocity
Four files over 2,700 lines each, three of them growing with every slice. This is not a style nit: it is the single biggest predictor of slowing slice cadence and merge-conflict pain. **Highest-leverage program-level fix:** introduce a *registry/dispatch* seam in each layer so a new slice registers a handler/panel/validator instead of editing the core file.
- Server: input-handler map keyed by intent type (`backend` report, refactor #1).
- Client: a `UIOrchestrator` child node + panel registry (`client` report, refactor #5).
- Tools: split `validate_shared.py` into protocol/rules/golden modules (`shared` report, improvement #5).

### Theme 2 — Conventions are enforced by prose, not by gates
Determinism (no `time.Now()`, no bare map ranges, seeded RNG only) is a **non-negotiable invariant per CLAUDE.md** — yet it is checked only by code review. Two bare `range s.equipped` violations already slipped in (`sim.go:3150`, `sim.go:5382`). A custom `go vet`/`staticcheck`/analysistest lint that fails CI on bare map iteration in `game/` would convert the project's most important invariant from hope to guarantee.

### Theme 3 — Documentation has drifted from a moving codebase
- `CLAUDE.md:13` says the project is at **v10**; it is at **v53** (43 slices stale). Agents are instructed to read this first.
- `protocol.go:2` package doc says **"20 Hz"**; the live loop is **15 Hz** (`protocol.go:17`). `CLAUDE.md` also says "fixed 20 Hz tick."
- `health_regen_per_second` vs `health_regen_per_10_seconds` — two names for one stat across rules/tooling/golden.
- No as-built architecture doc exists; ADR-0013 is "Proposed" but v51 already shipped it; ADRs 0002–0005 were promised by ADR-0001 and never written.

These are individually small and collectively corrosive: the docs an onboarding agent reads first are the ones most out of date.

### Theme 4 — Test coverage is integration-heavy, unit-light at the riskiest spots
Backend and bot coverage is strong. But the client's `_apply_delta` / `_apply_snapshot` / prediction-reconciliation — the most subtle client code — has **zero unit tests**. Golden fixtures lag slice delivery by ~4 slices (no goldens for v50–v53). Golden regeneration is fully manual, which is a correctness hazard on any formula refactor.

## 4. Top 10 recommendations (program-level, ranked by leverage)

1. **Fix `CLAUDE.md` staleness now** (1-line edit): v10 → v53, or replace the inline snapshot with a pointer to `PROGRESS.md`. Every agent session currently starts 43 slices behind. *(trivial / high)*
2. **Add a CI determinism lint** for `game/`: fail on `time.Now()`, non-seeded `rand`, and bare map ranges over sim state. Convert the core invariant from convention to gate. *(med / high)*
3. **Introduce the registry seam in `sim.go`**: extract a `map[InputType]handler` and move the ~24 `handle*` methods to `handlers.go`. New intents stop touching the core dispatcher. *(med / high)*
4. **Extract a shared GDScript `ItemRulesLoader` autoload + `ItemIconRenderer`**: kills the item-loader-×5 and icon-renderer-×3 duplication and 5 future edit sites. *(low / high)*
5. **Resolve the 15-vs-20 Hz and `health_regen` contradictions**, then add a `validate_shared.py` cross-check for stat-name consistency. *(low / med)*
6. **Write an as-built architecture doc** (`docs/ARCHITECTURE.md`): current module map, the *one* live protocol version, bot/validator roles, determinism contract. Reduces agent orientation cost on every task. *(med / high)*
7. **Add a `make regen-golden` path** that drives the Go sim to emit canonical fixtures; close the manual-edit correctness gap and backfill v50–v53. *(med / med)*
8. **Isolate bot/CI scaffolding** out of `main.gd` (~600–700 lines) and consider extracting the bot driver from `run.py` into importable modules. *(med / med)*
9. **Replace `panic(err)` in `NewSim`** (`sim.go:298`) with error propagation; a rules misconfig should not kill the process. Add a `default` arm + `push_warning` to the client `_handle_message` switch so unknown server messages aren't silently dropped. *(low / med)*
10. **Define a protocol-version deprecation policy** and archive v0–v5; today every on-disk version is validated as alive, scaling toward v15+ with no pruning. *(med / med)*

## 5. What "done well" looks like here (keep doing this)

- **Authoritative boundary is honored**: the client never computes an outcome that matters. No damage/loot/inventory authority leaked client-side (verified in `_apply_delta`).
- **Cross-language golden parity** is real infrastructure, not aspiration — Go tests and `test_golden.gd` both consume `shared/golden/`.
- **Determinism is taken seriously**: custom splitmix64 RNG, zero `time.Now()` in `game/`, pervasive `sorted*` iteration helpers, replay verification in CI.
- **SDD discipline**: 53 specs ↔ 53 plans ↔ 53 slices ↔ CI-green, with explicit non-goals per slice. This is the backbone that made 53 slices possible without collapse.
- **Lean dependencies**: Go server is gorilla/websocket + pgx + prometheus; no ORM, no reflection-heavy magic.

The foundation is excellent. The next phase of work should be **consolidation, not just accretion** — pay down the four monoliths and add the enforcement gates while the boundaries are still clean enough to make it cheap.
