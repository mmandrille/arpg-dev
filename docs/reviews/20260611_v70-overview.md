consumers.

## 3. Cross-cutting themes

### Theme 1 — v60 recommendations were acted on

Runtime sim construction no longer panics on default-world construction, and the content-library
manifest exists with backend/client loaders and validator tests. That is exactly how review
findings should flow into small slices.

### Theme 2 — Classes are now authoritative, but presentation is next

v69/v70 made class identity persistent and gameplay-relevant. The remaining user-facing work is
v71: class picker blocks, class sprites, class tooltips, and character-list class icons. Keep that
UI scoped to character selection and shared presentation data.

### Theme 3 — Defensive payload parsing is the next cheap safety win

The client and bot both still trust `equipped_update.slot`. The server owns payloads, but malformed
test or future protocol changes should not crash local consumers before diagnostics.

### Theme 4 — Large files are the ongoing tax

The system is still shippable, but repeated additions land in `main.gd`, `game_test.go`,
`validate_shared.py`, and `tools/bot/run.py`. Future slices should opportunistically add small
helpers when they touch those areas.

## 4. Top 10 recommendations
