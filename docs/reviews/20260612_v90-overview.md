## 3. Cross-cutting themes

### Theme 1 - Old correctness findings were paid down

The v80 fanout finding is closed: `server/internal/realtime/session_loop.go` snapshots connected client levels under lock before fanout. The v80 client payload finding is also closed: `client/scripts/main.gd` routes messages through a defensive `_envelope_payload` helper.

### Theme 2 - Data-driven skills need data-driven text

Skills, monsters, menus, settings, stats, shops, and future quest copy should resolve visible names through a shared text catalog. The current data-driven skill/visual direction is good, but localization must become part of the content contract rather than an afterthought in client scripts.

### Theme 3 - File splitting is now a standing rule

Current largest files are still significant: `server/internal/game/game_test.go` is 8619 lines, `server/internal/game/sim.go` is 7207, `client/scripts/main.gd` is 5470, `tools/bot/run.py` is 4915, and `tools/validate_shared.py` is 3110. Future slices should extract helpers whenever touching these areas.

### Theme 4 - Bot timing has improved but needs structure

The protocol bot elapsed budget is now 15 seconds and CI is green, including `teleporter_lab` at about 12.9 seconds. That is acceptable for today, but the long-term fix is per-scenario budgets or scenario partitioning rather than one global number.

## 4. Top 10 recommendations

1. Add an English localization catalog and require visible UI/content text to resolve by key.
2. Add Spanish translations plus a Settings language selector with English fallback.
3. Keep localization slices split: catalog/fallback first, selector/Spanish second.
4. Add validation for missing English keys and malformed locale files.
5. Migrate shared data presentation names through stable text keys, not duplicated literals.
6. Continue extracting touched client presentation code out of `main.gd`.
7. Split new server tests out of `game_test.go` by domain.
8. Split bot runner additions into modules instead of growing `tools/bot/run.py`.
9. Replace the global protocol-bot elapsed budget with per-scenario metadata when more long scenarios appear.
10. Keep `make skill-visual skill=all` as the visual regression path for data-driven skill presentation.
