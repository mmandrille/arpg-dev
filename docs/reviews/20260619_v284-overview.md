## 1. Executive summary

The v284 baseline is strong but not review-green. Since v250, the repo shipped expanded dungeon profiles, fog-of-war radius and LOS shadows, generated doors and door occlusion, discovery/minimap improvements, performance instrumentation, navigation budgets and movement LOD, class/monster model imports and rigging, bat dive and wolf pounce behavior, rectangle boss attacks, bat-summon boss phases, and a model-viewer tool.

Core architecture held. Gameplay facts remain server-owned: fog visibility filtering, closed-door occlusion, navigation budgets, monster attack reach/style, boss damage/summons, dungeon generation, and replayable protocol state all live in Go/shared rules. The Godot client renders those facts, exposes debug state for bots, and plays presentation-only animations from server events.

The main review finding is test-gate stability. Review-time `make ci` passed the early gates, protocol bot plus replay, and Godot smoke, but failed the full Godot client bot sweep with two scenarios: `mercenary_recovery_ui` and `door_fog_toggle`. Both passed focused local reruns, which points to timing/order sensitivity rather than deterministic feature breakage, but the official cadence gate failed and should be the first refactor target.

## 3. Cross-cutting themes

1. **Authority boundaries held through real gameplay additions.** Fog, doors, boss patterns, bat/wolf attack styles, and navigation budgets all stayed server/shared-rule owned.
2. **Presentation work is increasingly inspectable.** Fog, minimap, boss telegraphs, performance status, and model viewer state are exposed through debug surfaces and covered by client tests or bot scenarios.
3. **The maintainability ratchets are working.** Helper-global debt is at zero, and file-size pressure is visible enough to steer future splits.
4. **Full-suite client bot stability is now the top process risk.** The two review-time failures passed focused reruns, so the likely issue is scenario timing/order/state synchronization in the broad sweep.
5. **Asset work is maturing, but provenance needs a production pass.** The manifest and validators are good; several user-provided class models still have unverified license metadata.
