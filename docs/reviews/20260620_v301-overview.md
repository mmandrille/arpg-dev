## 3. Cross-cutting themes

1. **The World Detail/Navigation queue landed without authority drift.** The server remains the
   source for movement, pathfinding, obstacle kinds, LOS blockers, and fog visibility.
2. **The bot catalogs are valuable but need budget hygiene.** The full bot sweep caught residual
   timing/precision problems; small scenario-runner and JSON budget fixes restored green CI.
3. **Focused client presentation factories are working.** v301 added material depth without touching
   `main.gd`, gameplay state, or protocol contracts.
4. **Worktree isolation exposed local infrastructure coupling.** The fixed `arpg-postgres` Compose
   container name made final CI require `COMPOSE_PROJECT_NAME=arpg-dev`.
5. **Production asset readiness still needs provenance cleanup.** The validation pipeline is strong,
   but user-provided GLB licenses remain unverified in the asset manifest.

## 4. Top 10 recommendations

1. Make `make db-up` / `make ci` worktree-safe without hidden `COMPOSE_PROJECT_NAME` knowledge.
2. Add `go vet ./...` to `make ci` while it is green.
3. Split the next touched generated-dungeon subdomain out of `server/internal/game/dungeon_gen.go`.
4. Split the next touched `main.gd` bot/debug-state aggregation into a focused helper.
5. Split the next touched backend assertion cluster out of `server/internal/game/game_test.go`.
6. Extract the next touched assertion domain from `client/scripts/bot_assertion_handlers.gd`.
7. Add a bot-scenario authoring note or validator hint for long generated-route budgets and float
   bound tolerance.
8. Split the next touched validation domain out of `tools/validate_shared.py`.
9. Resolve or replace `user-provided-unverified` GLB provenance before production distribution.
10. Keep production dungeon art/VFX and richer obstacle material work as explicit future visual
    slices.
