## 1. Executive summary

The v302–v308 batch is the **strongest data-driven, boundary-respecting feature batch in the repo's
history**, and also the one that **most clearly let process discipline slip**. Seven world-detail and
navigation features shipped — water/hole hazards, an obstacle variety pack, a per-monster flying
navigation trait, barbarian leap obstacle-crossing, and line-of-sight blockers that gate fog reveal.
Every one is genuinely server-authoritative: the client renders state and plays presentation, while
collision, pathability, occlusion truth, and leap landing all come from the server. Every new
balance-sensitive value is in `shared/rules/` behind a tight schema (`worlds` +194, `dungeon_generation`,
`monsters` flying trait, `skills` leap), and the new mechanics are covered by rule-derived tests, a real
deterministic obstacles golden, and four meaningful bot scenarios. All component gates are green:
`validate-shared` (1,435 checks), `validate-assets` (121), `lint-determinism`, `go test`, `go vet`,
`pytest` (137), `client-unit`, and `make maintainability`.

The cost shows up in three softer dimensions. **Documentation drifted:** `docs/CODEMAP.md` — the
documented "load these files first" index — was never updated for any of the new backend or client
files, and `validate_codemap.py` cannot catch the omission. **Test wiring regressed:** five focused unit
tests (two from this batch, three from v301) exist on disk but are never registered in
`client_smoke.sh`, so they run zero times in CI — the v301 review even cited the v301 ones as coverage
they do not provide. **Maintainability headroom was spent:** the batch pushed `rules.go` and `types.go`
to *exactly* their +25 ratchet caps and six client files toward theirs, without lowering a single
baseline. None of these is a correctness bug, but together they are exactly the kind of debt a periodic
review exists to surface before it compounds — and they map cleanly onto the `$refactor` queue.

All four prior v301 recommendations that were actionable were checked: worktree-safe CI and the
presentation-feel config owner are **RESOLVED**; the validator/coordinator-split recs and the GLB
provenance item remain **OPEN**.

## 3. Cross-cutting themes

1. **Architecture/data discipline is now a genuine strength.** Server authority and rules-as-data held
   under a feature-heavy batch — every specialist scored boundaries 8–9 with cited, server-owned outcome
   paths. This is the thing to protect.
2. **`docs/CODEMAP.md` staleness is the single most-cited finding** — all three specialists flagged it
   independently. The index that tells agents which files to load is wrong for the headline batch, and
   `validate_codemap.py` only checks that listed paths exist, not that present files are listed, so CI is
   blind to it. A one-line generator/validator strengthening closes the whole class.
3. **The maintainability ratchet is doing its job but is being treated as a budget to spend.** Multiple
   grandfathered coordinators sit at or one line from their caps with no baselines lowered. The ratchet
   policy explicitly warns the +25 allowance is for incidental drift, not routine growth.
4. **Test *methodology* is excellent; test *wiring/policy* is not.** Rule-derived expectations, semantic
   ranges, deterministic goldens, no accidental tuning pins — but five unit tests don't run, and a
   routinely-ignored rule #9 should be either enforced or amended.
5. **Governance docs have started to contradict the code** (rule #12 10s vs run.py 15s; "cross-language"
   golden that is Go-only; combat-feel constants without the required code-ownership note). Cheap to
   reconcile, corrosive if left.

## 5. What "done well" looks like here

- `docs/CODEMAP.md` lists every server/client domain file, and `validate_codemap.py` fails CI on an
  unlisted-but-present file so the index can never silently rot again.
- `make client-unit` runs every unit test in `client/tests/` (no orphan files), and rule #9 is either
  enforced or explicitly amended so the test suite and the policy agree.
- The maintainability ratchet trends *down*: at-cap coordinators (`rules.go`, `types.go`,
  `bot_scenario_runner.gd`) get extractions that lower their baselines, not edits that spend the +25.
- Governance docs match the code: one budget number, accurate golden framing, and an explicit note
  wherever a tuning value is intentionally code-owned.
- The architecture/data-driven discipline that made this batch strong stays the default for the next
  feature batch, with new world-model decisions recorded in an ADR as they ship.
