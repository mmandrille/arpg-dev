## Summary

The backend improved materially since v60. The highest prior backend finding is resolved:
`NewSim` now returns `(*Sim, error)` and the panic wrapper is isolated in `MustNewSim` for tests
(`server/internal/game/sim.go:322`). v70 also made class identity authoritative in the sim and
handlers: cross-class weapon equips reject at `server/internal/game/handlers.go:671`, skill spends
reject at `server/internal/game/handlers.go:1076`, and casts reject at
`server/internal/game/handlers.go:1122`.

The main remaining backend risk is unchanged from v60: realtime fanout still reads sim player levels
after the loop mutex is released. The system remains correct under the current single-loop model,
but this should be closed before more reconnect/background session behavior is added.

## 1. Architecture

- **[Strength] Server authority remains crisp.** Class requirements are enforced server-side before
  mutation (`handlers.go:671`, `handlers.go:1076`, `handlers.go:1122`), and shared validation backs
  the data (`tools/validate_shared.py:836`).
- **[Strength] The v60 panic-constructor finding is resolved.** `NewSim` returns errors and
  `MustNewSim` documents test-only panic behavior at `server/internal/game/sim.go:322`.
- **[Med] Realtime fanout still reads sim state after unlock.** The tick loop unlocks at
  `server/internal/realtime/session_loop.go:514`, then `fanoutResult` calls
  `l.sim.PlayerCurrentLevel` at `server/internal/realtime/session_loop.go:529`. Snapshot client
  levels while locked and pass them into fanout.
- **[Strength] Content-library loading has a real backend seam.** The skills manifest loader rejects
  empty files, duplicate skill IDs, absolute paths, and bad manifest shape in
  `server/internal/game/rules_manifest.go:34`.

## 2. Technical

- **[Strength] v70 class gates are covered at the gameplay boundary.** The new tests assert
  class-specific spend/cast and wrong-class weapon rejection around
  `server/internal/game/game_test.go:817`.
- **[Strength] Replay and realtime carry class identity.** v70 adds class to progression views and
  session-start progression, and the websocket Magic Bolt test now creates a sorcerer character
  before proving the skill path.
- **[Low] Invariant panics remain, but they are narrower.** `activeLevel` still panics if the
  internal level map is corrupt (`server/internal/game/sim.go:1688`). That is acceptable as an
  invariant guard, unlike the old runtime constructor panic.
- **[Med] `game_test.go` keeps absorbing new sim behavior.** At 7,896 LOC, it is still the backend
  test sink. New loader, class, and equipment tests should move into focused files when they do not
  require broad sim fixtures.

## 3. Maintainability

- **[Strength] `rules_manifest.go` avoided growing `rules.go` for the v60 manifest follow-up.**
  This is the right direction for future content indexes.
- **[Med] `rules.go` and `validate_shared.py` are still pressure points.** `rules.go` is 2,686 LOC
  and the validator cross-check function starts at `tools/validate_shared.py:196`; class/item checks
  continue to land in the same long function.
- **[Low] Store repository breadth is increasing.** `server/internal/store/repos.go` is 2,190 LOC
  and now owns sessions, progression snapshots, stash, market, shop stock, and class snapshot rows.
  Future market purchase work should consider focused repository files.

## 4. Documentation

- **[Strength] v70 has spec, plan, as-built, and progress entries.**
- **[Med] Package-level docs for the deterministic sim are still thin.** ADR-0001 records the
  determinism contract, but the `game` package itself does not state it at the source boundary.

## Top 5 backend refactors

1. Snapshot player levels under `sessionLoop` lock before `fanoutResult`.
2. Split future sim tests into focused files by subsystem instead of growing `game_test.go`.
3. Extract shared validator sections for class/items/content into helper modules.
4. Keep new content loaders in dedicated files like `rules_manifest.go`.
5. Add a `game` package doc covering determinism, authoritative ownership, and allowed panic scope.

*Evidence: direct reads of current server code, prior v60 backend review, ADR-0001, v70 tests/docs;
`make ci`, `go vet ./...`, and focused Go package tests passed.*
