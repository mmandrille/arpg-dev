## Summary

The backend made the highest-value v140 paydown: market persistence moved out of the broad store coordinator, and session load/player lifecycle code moved out of `sim.go`. The server-authoritative boundary remains intact. The main backend risk is now less about market correctness and more about the remaining test/sim monoliths: `game_test.go` is still the largest file in the repo, and `sim.go` still owns many gameplay domains even after the v142 extraction.

## 1. Architecture

[Strength] Market persistence now has a focused home. Active listing reads expire stale rows before returning state, and listing creation, cancellation, offer creation, offer cancellation, acceptance, expiration, audit, and summary logic live in `server/internal/store/market_repo.go` instead of the general repository file (`server/internal/store/market_repo.go:11`, `server/internal/store/market_repo.go:40`, `server/internal/store/market_repo.go:96`, `server/internal/store/market_repo.go:151`).

[Strength] Market ownership transitions still happen transactionally. Publish locks and deletes the seller stash item, offer creation locks and deletes bidder stash items, and both paths write audit rows inside the same transaction (`server/internal/store/market_repo.go:45`, `server/internal/store/market_repo.go:73`, `server/internal/store/market_repo.go:80`, `server/internal/store/market_repo.go:156`, `server/internal/store/market_repo.go:185`, `server/internal/store/market_repo.go:198`).

[Strength] Session load and player lifecycle now have domain files. Persisted inventory/hotbar/skill/stash loading lives in `sim_load.go` (`server/internal/game/sim_load.go:37`, `server/internal/game/sim_load.go:59`, `server/internal/game/sim_load.go:81`, `server/internal/game/sim_load.go:131`), while co-op player lifecycle, player metadata, player lookup, respawn, and spawn selection live in `sim_players.go` (`server/internal/game/sim_players.go:11`, `server/internal/game/sim_players.go:87`, `server/internal/game/sim_players.go:134`, `server/internal/game/sim_players.go:170`, `server/internal/game/sim_players.go:204`).

[Med] `repos.go` is smaller but still a mixed persistence coordinator. It now starts with accounts and character lifecycle (`server/internal/store/repos.go:21`, `server/internal/store/repos.go:51`), but still also carries several durable session/progression concerns. Future persistence domains should follow the market split rather than adding another block here.

## 2. Technical

[Strength] Determinism-sensitive code moved without changing the live/replay contract. The extracted player helpers keep sorted player iteration for character lookup and player IDs (`server/internal/game/sim_players.go:134`, `server/internal/game/sim_players.go:142`), and v142/v149 closeout ran full CI including determinism lint and replay.

[Strength] Market expiration freshness from v139 survived the extraction. `ListActiveMarketListings` still calls `ExpireMarketListings` before querying active rows (`server/internal/store/market_repo.go:11`), preserving the stale-read fix while changing file ownership.

[Med] `sim.go` remains a high-blast-radius file at 7045 lines. v142 removed load/player lifecycle, but combat, tick orchestration, dungeon/event logic, and many helper paths still share one coordinator. The next sim extraction should target a cohesive gameplay domain that already has focused tests, not a broad mechanical move.

## 3. Maintainability

[Strength] The backend trend moved down materially since v140: `repos.go` 3058 -> 2315, `sim.go` 7801 -> 7045, and the grandfathered total 68778 -> 65592 lines.

[High] `game_test.go` is now the clearest backend debt. It remains 9116 lines, larger than `sim.go`, and acts as a default landing zone for unrelated gameplay tests. The code is covered, but local reasoning and review scope suffer when new tests are added there by habit.

[Med] `rules.go` remains at 3262 lines. It is stable, but rule-loading additions should prefer catalog-specific validators/loaders when possible, especially for new economy or progression catalogs.

## 4. Documentation

[Strength] CODEMAP now points market work at focused store files, and session/realtime work at `sim_load.go` and `sim_players.go`, reducing future broad-grep pressure (`docs/CODEMAP.md:7`, `docs/CODEMAP.md:17`).

[Low] The review cadence is working: v140 recommendations directly produced v141 and v142 backend paydown, and the current review records the trend.

## Top 5 backend refactors

1. Split the next cohesive `sim.go` domain, preferably tick/event orchestration or combat helpers with focused replay/game tests.
2. Drain `game_test.go` opportunistically into domain test files when touching combat, dungeon, market, stash, or class behavior.
3. Continue moving persistence domains out of `repos.go` only when a complete domain can move with its helpers and tests.
4. Keep market audit writes inside the same transaction as every future market ownership mutation.
5. Avoid growing `rules.go` for new catalogs; add catalog-specific loaders/validators when the domain is non-trivial.
