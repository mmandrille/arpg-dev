## Summary

The backend is not review-green at v200. The highest-risk regression is shared skill rule drift that makes established Barbarian and Ranger skill expectations fail in Go tests and protocol scenarios. The repository still preserves many core backend strengths: deterministic lint passes, shared validation passes, most protocol scenarios pass, and companion/mercenary scenarios are mostly healthy. But the current branch cannot be treated as shippable until the skill tree/rule rows and CI gate are repaired.

## 1. Architecture

- **[Strength] Server authority is still the gameplay boundary.** Companion and mercenary entities are still advanced by Go simulation code, and the client consumes state/events rather than deciding combat results.
- **[High] Skill-tree rule drift breaks class progression semantics.** `rage` now requires `piercing_shot` in shared data (`shared/rules/skills.v0.json:362`), which conflicts with Barbarian tests expecting Rage to be spendable for a Barbarian baseline (`server/internal/game/game_test.go:947`). The Ranger third-skill relationship also drifted: `volley` requires `split_arrow` (`shared/rules/skills.v0.json:868`) while `split_arrow` is still tier 2 and requires `piercing_shot` (`shared/rules/skills.v0.json:955`, `shared/rules/skills.v0.json:971`), conflicting with tests that expect Split Arrow as the tier-3 Volley follow-up (`server/internal/game/class_third_skills_test.go:17`, `server/internal/game/game_test.go:258`).
- **[Med] Companion and elite-minion AI tuning remains code-owned.** Follow/assist constants live in Go (`server/internal/game/companion_ai.go:5`) and are reused by elite-minion movement (`server/internal/game/elite_minion_ai.go:45`, `server/internal/game/elite_minion_ai.go:61`). This was a v190 recommendation and remains open.

## 2. Technical

- **[High] Go tests are red.** `make ci` failed `go test ./...` in `server/internal/game`: `TestThirdClassSkillsRequirePrerequisites`, `TestLoadRules`, `TestClassSkillAccessGatesSpendabilityAndLearning`, and `TestRageBuffAppliesStatsVisualScaleAndExpires` all fail against current rule data.
- **[Strength] Determinism and contract validation still pass.** CI passed extraction-coupling ratchet, shared schema validation, asset validation, determinism lint, Python tests, and the final Godot smoke suite.
- **[Med] Monster-rarity validation still mixes tuning ownership into the runtime loader.** `validateMonsterRarities` hardcodes rarity ids, weights, offsets, colors, and visual scales in Go (`server/internal/game/rules.go:2281`). This remains a future golden/schema ownership cleanup.
