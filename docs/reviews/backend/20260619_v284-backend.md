## 1. Architecture

- **[Strength] Fog visibility is server-authored for gameplay facts.** Recipient snapshots and deltas go through `visibleEntityViewsForPlayer`, `FilterChangesForPlayer`, and `FilterEventsForPlayer`, hiding living monsters outside radius or behind occluders (`server/internal/game/fog_of_war.go:9`, `server/internal/game/fog_of_war.go:45`, `server/internal/game/fog_of_war.go:68`). Closed barrier interactables now participate in the same LOS block path (`server/internal/game/fog_of_war.go:219`).
- **[Strength] Dungeon generation kept deterministic, rule-owned expansion.** `GenerateDungeonLevel` derives per-level RNG streams from the session seed and level number rather than consuming live sim RNG (`server/internal/game/dungeon_gen.go:13`), and generated doors are committed only after reachability succeeds (`server/internal/game/dungeon_gen.go:215`).
- **[Strength] Navigation load shedding is server-owned.** Monster path requests and node visits are bounded by active navigation rules (`server/internal/game/monster_navigation_budget.go:8`, `server/internal/game/monster_navigation_budget.go:21`), movement LOD is deterministic by monster id/tick (`server/internal/game/monster_movement_lod.go:29`), and overload degradation is a transient sim deadline (`server/internal/game/monster_overload_guardrails.go:3`).
- **[Strength] New monster and boss behaviors stay authoritative.** Dive/pounce attack styles are emitted only from server combat events and ordinary melee omits the style (`server/internal/game/monster_attack_style_test.go:25`). Rectangle boss hit testing requires locked aim and positive width (`server/internal/game/boss_rectangle_pattern_test.go:5`), while boss summons spawn authoritative monsters and emit `boss_summoned_adds` once (`server/internal/game/boss_patterns.go:216`, `server/internal/game/boss_summon_pattern_test.go:50`).
