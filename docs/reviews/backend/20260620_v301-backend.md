## Summary

The backend is green and stronger than the v284 review baseline. v295-v300 added water, holes,
flying navigation exceptions, Barbarian Leap obstacle crossing, obstacle variety, and line-of-sight
blocker semantics while keeping collision, pathfinding, fog filtering, and replay facts on the Go
side. v301 was client presentation work, but the final batch CI also stabilized two server-side test
residuals: deterministic LOS metadata pointer identity for generated obstacles and a generated-door
population test seed that still emits generated doors.

The main backend concern remains maintainability scale, not architecture correctness. The largest Go
coordinators and assertion files are still well over the ratchet target, so future backend work
should continue extracting the next touched generation, rules, and test domains.

## 1. Architecture

- **[Strength] Navigation and obstacle authority remain server-owned.** The World
  Detail/Navigation queue added obstacle kinds and traversal exceptions without moving movement or
  collision decisions to the client. The client receives wall layout metadata; Go still owns
  reachability, fog filtering, and replayable state.
- **[Strength] Generated obstacle LOS metadata is deterministic after v301 residual stabilization.**
  `solidObstacleLineOfSightOverride` returns the shared true pointer for rock/column kinds
  (`server/internal/game/dungeon_obstacle_variety.go:12`,
  `server/internal/game/dungeon_obstacle_variety.go:53`), so generated wall comparisons no longer
  fail on pointer address differences.
- **[Strength] Generated-door coverage still proves Sim population, not just raw generation.**
  `TestGeneratedDungeonDoorsPopulateAsClosedInteractables` now drives a seed known to produce
  generated doors and verifies closed interactable entities in level state
  (`server/internal/game/dungeon_doors_test.go:45`).

## 2. Technical

- **[Strength] The official gate is green.** `COMPOSE_PROJECT_NAME=arpg-dev make ci` passed in
  11m50s at the v301 baseline, including `go test ./...`, protocol bot plus replay, all 81 Godot
  client bot scenarios, and Godot headless smoke.
- **[Strength] Extra backend static checking is green.** `cd server && go vet ./...` passed during
  this review.
- **[Med] CI does not run `go vet` yet.** `go vet` is cheap and currently clean, but it is outside
  `scripts/ci.sh`; adding it would keep the review-time backend check from being a manual-only
  proof.

## 3. Maintainability

- **[Strength] The extraction-coupling ratchet remains at zero.** `make maintainability` reports
  0 helper-global coupling occurrences.
- **[Med] Large backend files remain the main backend drag.** `game_test.go`, `sim.go`, `rules.go`,
  `repos.go`, `store_test.go`, and `dungeon_gen.go` remain large. The file-size ratchet is doing its
  job, but the next touched backend domain should reduce or avoid these files instead of adding
  more inline branches.
- **[Low] The generated obstacle LOS true pointer is pragmatic but subtle.** It fixes deterministic
  struct comparisons, but future code should avoid mutating pointer-owned optional metadata and
  prefer semantic comparison helpers if this pattern spreads.

## 4. Documentation

- **[Strength] v295-v301 lifecycle rows now record the green batch CI result.**
  `docs/progress/slice-lifecycle.md:310` through `docs/progress/slice-lifecycle.md:316` show the
  selected World Detail/Navigation queue as focused-check plus batch-CI green.
- **[Low] Old v285-v293 lifecycle rows still preserve historical pending/red wording.** v294

## Top 5 backend refactors

1. Add `go vet ./...` to the normal CI gate while it is green and cheap.
2. Split the next touched generated-dungeon subdomain out of `server/internal/game/dungeon_gen.go`.
3. Split the next touched backend assertion cluster out of `server/internal/game/game_test.go`.
4. Keep new obstacle/fog/pathfinding metadata comparisons semantic rather than pointer-address
   dependent.
5. Continue keeping gameplay tuning in shared rules and tests rule-derived when future navigation or
   obstacle balance changes land.
