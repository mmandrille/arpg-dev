## Summary

The authoritative server boundary remains intact: intents dispatch through `inputHandlers`, production wiring is `hub` → `sessionLoop` only, and the v370–v384 perf batch added **focused extractions** (`tick_collision_cache.go`, `combat_tick_budget.go`, `persist_defer.go`, `tick_guardrails.go`) without protocol churn. `sim.go` shrank ~78 lines since v349 while gaining collision cache and overload state.

The v337 refactor queue is **mostly open**: dead `runner.go` (~650 lines, zero call sites) now **diverges further** from production (no persist defer, no `TickResultsProfiled`), `simTickCtx` paydown stalled, and `advanceMonsterMovementBudgeted` duplicates `advanceMonsterMovement`. v370–v384 backend as-builts are stub-only — behavioral contracts for defer triggers and overload policy are undocumented.
