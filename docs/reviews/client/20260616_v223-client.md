## Summary

The client remains thin and green at v223. Full CI passed 47/47 client bot scenarios plus the Godot smoke suite. The v219-v223 feature batch added player-visible UI for companion stance, resource wallet-backed upgrades, upgrade previews, and Bloodbound Sigil in the unique chest without moving authority into the client.

The most meaningful client improvement since v200 is maintainability direction: `main.gd` dropped from 6,521 lines in the prior review to 5,760 lines now, helped by focused scripts such as `boss_visuals_controller.gd`, `boss_visuals_context.gd`, and `blacksmith_upgrade_preview.gd`. It is still far above the 600-line target and remains the client risk center.

## 1. Architecture

- **[Strength] Client remains a renderer/input layer.** `main.gd` explicitly documents that the server owns combat, loot, and inventory outcomes (`client/scripts/main.gd:1`), and recent UI features consume protocol/shared-rule state rather than computing authoritative outcomes.
- **[Strength] Companion bar click handling is localized.** The top-left companion block now emits `companion_selected` from `CompanionBar` (`client/scripts/companion_bar.gd:4`, `client/scripts/companion_bar.gd:146`), keeping the visual slot behavior out of server logic.
- **[Strength] Blacksmith preview logic is extracted and pure.** `BlacksmithUpgradePreview.preview_lines` computes display text from item/context dictionaries (`client/scripts/blacksmith_upgrade_preview.gd:31`), preserving server ownership of actual upgrade outcome.
- **[Med] `main.gd` is still the broad presentation router.** It preloads and wires many UI/domain scripts (`client/scripts/main.gd:21`, `client/scripts/main.gd:45`, `client/scripts/main.gd:61`) and still owns entity reaction, projectile, interactable, and bot facade routing (`client/scripts/main.gd:1707`, `client/scripts/main.gd:5078`, `client/scripts/main.gd:5097`, `client/scripts/main.gd:5520`).

## 2. Technical

- **[Strength] Client verification is green.** Full CI passed all 47 client bot scenarios and Godot headless smoke in the final 8m25s run.
- **[Strength] Boss and blacksmith presentation now have focused helpers.** `BossVisualsController` owns boss health/phase display synchronization (`client/scripts/boss_visuals_controller.gd:17`, `client/scripts/boss_visuals_controller.gd:100`), and blacksmith preview text is covered by both unit and client bot proof.
- **[Low] Direct presentation fallbacks still exist by necessity.** Some client paths still provide local display defaults for testability and resilience; future feature code should keep those as presentation fallbacks only, not gameplay facts.

## 3. Maintainability

- **[Strength] The v215/v216 split campaign moved in the right direction.** `main.gd` is now 5,760 lines, down from 6,521 in v200, with extracted context/controller files present for boss visuals and focused factory-style logic.
- **[Med] Large client files remain managed debt.** `.maintainability/file-size-baseline.tsv` still grandfathers `client/scripts/main.gd` (`.maintainability/file-size-baseline.tsv:7`), `inventory_panel.gd` (`.maintainability/file-size-baseline.tsv:6`), `test_coop_client.gd` (`.maintainability/file-size-baseline.tsv:14`), and several panel/bot files.
- **[Low] Client bot runner size continues to grow slowly.** `client/scripts/bot_scenario_runner.gd` is 1,674 lines and should keep delegating new steps into action/wait/assertion helper scripts.

## 4. Documentation

- **[Strength] Visual proof is named and runnable.** Recent as-built notes consistently name `make bot-visual scenario=mercenary_roster_ui`, `blacksmith_upgrade_ui`, and `unique_chest_client_proof` where visual inspection is useful.
- **[Med] Progress archive links still hurt navigation from client review context.** `docs/progress/slice-lifecycle.md:20` links to `docs/specs/...` from inside `docs/progress/`, which resolves incorrectly in Markdown renderers that treat links as file-relative.

## Top 5 client refactors

1. Continue extracting the next touched presentation path from `client/scripts/main.gd`, prioritizing bot facade routing or entity reaction/projectile presentation.
2. Keep companion UI additions inside `companion_bar.gd`, `mercenary_panel.gd`, or focused bridge scripts.
3. Split `client/tests/test_coop_client.gd` before adding more co-op/companion assertions.
4. Add or fix progress-link validation so moved archive links no longer mislead agents.
5. Keep blacksmith/resource/unique UI proof display-only and backed by server/protocol assertions.
