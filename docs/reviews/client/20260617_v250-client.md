## Summary

The client remains a renderer and input layer at v250. The batch added market search/sort,
expiration labels, market item comparison, the material wallet window, blacksmith recipe/history
presentation, mercenary combat stat display, a boss reward panel, and shape-specific boss telegraph
decals. None of those changes moved combat, market, upgrade, item, or companion authority into
GDScript.

Client maintainability improved where new helpers were introduced, but the usual large files remain
the main cost. `main.gd` stayed line-neutral at 5,760 lines while boss presentation stayed in
`BossVisualsController`; `market_panel.gd`, `shop_panel.gd`, `inventory_panel.gd`, and large client
tests remain managed debt.

## 1. Architecture

- **[Strength] Thin-client boundary is explicit and still respected.** `main.gd` documents the
  renderer/input split and server ownership of combat, loot, and inventory outcomes
  (`client/scripts/main.gd:1`). Recent batch work renders server state or emits existing HTTP/WS
  requests rather than computing outcomes locally.
- **[Strength] Market work used focused presentation helpers.** Market row title/expiration debug
  state lives in `MarketListingRows` (`client/scripts/market_listing_rows.gd:42`,
  `client/scripts/market_listing_rows.gd:71`), filtering/sorting lives in `MarketRowFilters`
  (`client/scripts/market_row_filters.gd:1`), and comparison text lives in `MarketItemComparison`
  (`client/scripts/market_item_comparison.gd:1`, `client/scripts/market_item_comparison.gd:72`).
- **[Strength] Boss presentation stayed display-only.** `boss_killed` maps to a HUD reward message
  (`client/scripts/main.gd:1264`) and `BossVisualsController` builds decals from existing telegraph
  metadata (`client/scripts/boss_visuals_controller.gd:177`, `client/scripts/boss_visuals_controller.gd:221`).
  The client exposes `telegraph_marker_shape` only for presentation/debug proof
  (`client/scripts/main.gd:5408`).
- **[Med] Client debug progression seeding lacks a focused unit proof.** v249 added full JSON
  debug progression for client bots (`scripts/bot_client.sh:264`, `client/scripts/bot_debug_progression_setup.gd:24`)
  and integer-normalizes nested maps before the debug endpoint call (`client/scripts/net_client.gd:152`,
  `client/scripts/net_client.gd:172`). It is covered by boss client scenarios, but a direct
  `NetClient`/setup unit would make this tooling contract cheaper to guard.

## 2. Technical

- **[Strength] Client batch proof is broad.** The final v241-v250 `make ci` ran the full Godot
  client bot suite with 66 passing scenarios, including v241-v250 proofs. Focused slice checks also
  covered market helpers, blacksmith panel behavior, mercenary panel behavior, boss health bar
  reward state, factory telegraph decals, and client bot step loading.
- **[Strength] Boss telegraph decals are inspectable by bots.** `66_boss_telegraph_decals.json`
  asserts `line`, `summon_circle`, and `cone` marker shapes through entity presentation debug
  rows (`tools/bot/scenarios/client/66_boss_telegraph_decals.json:33`,
  `tools/bot/scenarios/client/66_boss_telegraph_decals.json:43`,
  `tools/bot/scenarios/client/66_boss_telegraph_decals.json:53`).
- **[Low] Code-native telegraph meshes are good enough for now but not final production VFX.**
  ADR-0009 expects readable telegraph-first boss mechanics; production floor decals, particles, or
  exact line/cone aim projection remain future presentation work.

## 3. Maintainability

- **[Strength] New boss logic did not grow `main.gd`.** Boss health/phase/reward/decal behavior is
  routed through `BossVisualsController` and its context (`client/scripts/main.gd:3373`,
  `client/scripts/main.gd:3377`), while `main.gd` remains a broad coordinator under the ratchet.
- **[Strength] Market helpers reduced panel pressure.** `market_panel.gd` now delegates title,
  expiration, comparison, and filtering behavior to focused helpers (`client/scripts/market_panel.gd:488`,
  `client/scripts/market_panel.gd:513`, `client/scripts/market_panel.gd:736`).
- **[Med] Large client files remain the highest client-side maintenance cost.** `main.gd`,
  `bot_scenario_runner.gd`, `inventory_panel.gd`, `shop_panel.gd`, `stash_panel.gd`,
  `skills_panel.gd`, and several client tests are still above the 600-line target.
- **[Low] Market/shop/inventory comparison code should not drift into parallel implementations.**
  The market and shop panels both render comparison counts/lines; future refactors should share
  lower-level display helpers when this area is touched again.

## 4. Documentation

- **[Strength] Visual proof commands are present.** v241-v250 as-built files name the relevant
  `make bot-visual scenario=...` command for manual visual inspection where the slice changed UI.
- **[Low] Per-slice proof docs still say batch CI is pending.** This should be corrected now that
  the selected batch `make ci` passed after v250.

## Top 5 client refactors

1. Add a focused GDScript unit for client debug progression JSON parsing and `NetClient._int_value_map`.
2. Close the v241-v250 as-built/lifecycle batch-CI wording.
3. Extract the next touched `main.gd` debug/presentation-row path instead of adding more keys inline.
4. Split future market/shop comparison display duplication when either panel is next touched.
5. Keep production telegraph VFX or exact projected aim decals as a planned feature, not incidental refactor.
