# v155 Spec - Random Quest Reward Floors

Status: Draft
Date: 2026-06-14
Codename: `random-quest-reward-floors`

## Purpose

Begin autogenerated dungeon quests with a small server-owned floor roll: roughly 10% of generated
non-boss dungeon floors become quest reward floors and place an extra reward chest. Opening that
chest uses the existing authoritative chest and loot path.

## Non-goals

- No town NPC dialog, quest log UI, durable quest persistence, or multi-objective quest system.
- No new protocol schema version.
- No client presentation beyond existing chest/loot rendering.
- No final reward economy tuning.

## Acceptance criteria

1. Generated non-boss dungeon floors deterministically roll quest reward floor eligibility from
   session seed and depth.
2. The roll is approximately 10% across a representative depth range.
3. Eligible floors place an extra reachable reward chest using existing chest/open/loot mechanics.
4. Boss floors are excluded from this random quest-floor roll.
5. Focused Go tests prove determinism, 10% distribution bounds, boss exclusion, and chest opening.
6. Protocol bot proof descends to a seeded quest reward floor, opens the reward chest, and observes
   loot.

## Scope and likely files

- `server/internal/game/dungeon_gen.go`
- focused Go test file
- `tools/bot/scenarios/65_random_quest_reward_floor.json`
- `PROGRESS.md`, plan, and as-built docs

## Open questions and risks

- This is intentionally the first quest-generation slice, not the final quest system. Quest log,
  objective types, and durable progress remain deferred.
